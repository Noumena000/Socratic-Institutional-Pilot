# Phase 1 Acceptance Criteria

Phase 1 is complete when the public demonstration can show the institutional workflow without requiring a backend or exposing the private reasoning engine.

## Required experience

- A faculty setup view identifies the course, objective, evidence requirements, and knowledge-pack boundary.
- A learner view presents a bounded assessment and fictional response.
- A multi-turn Socratic dialogue can be advanced interactively.
- A “Why this question?” panel explains the objective, observed evidence, missing evidence, and pedagogical move.
- Evidence indicators update across the dialogue.
- A faculty evidence portfolio distinguishes demonstrated understanding from unresolved evidence.
- A provider-boundary view explains that the language model is replaceable.
- The demonstration can be reset and replayed.

## Governance boundaries

- All data is fictional.
- No provider endpoint or credential is present.
- No automatic grade or misconduct conclusion is generated.
- Faculty review is explicit.
- The private reasoning implementation remains absent.

## Technical boundaries

- Runs as static HTML, CSS, and JavaScript.
- Works without package installation.
- Supports GitHub Pages deployment from the repository root.
- Uses semantic controls, keyboard-visible focus, and reduced-motion support.
